# Mac Local Dictation (SuperWhisper Clone)

Полностью локальное приложение для голосового ввода (диктовки) на macOS. Использует `mlx-whisper` (на базе Apple Silicon GPU) для идеального распознавания Рунглиша и любых языков без интернета.

Приложение работает в фоновом режиме (иконка в статус-баре) и имитирует естественный ввод с клавиатуры.

## Особенности
- **Локально и приватно**: Никакие данные не уходят в облако.
- **Hold-to-Talk или Toggle**: Зажмите `Cmd + D` (или нажмите дважды), надиктуйте текст и отпустите. Текст сам напечатается в активном окне.
- **Нативный ввод Swift**: Приложение не просто вставляет текст из буфера обмена (хотя и копирует его туда для надежности), а использует нативный `CGEvent` macOS для эмуляции нажатий клавиш. Это позволяет использовать диктовку в полях ввода, где `Cmd+V` заблокирован или работает криво.
- **Защита от MLX Metal OOM**: Встроен обход известного бага `SIGABRT` в старых версиях MLX. Процесс транскрипции изолирован мьютексом (`threading.Lock()`), а память GPU принудительно очищается (`mlx.core.clear_cache()`) после каждой фразы.

## Требования
- Mac на Apple Silicon (M1/M2/M3/M4 и т.д.)
- Python 3.9+
- Установленные `ffmpeg` и `sox` (для записи звука: `brew install sox`)

## Установка

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/YOUR_USERNAME/mac-local-dictation.git
   cd mac-local-dictation
   ```

2. Запустите скрипт установки (скомпилирует Swift-хелпер и установит PIP-пакеты):
   ```bash
   ./build_and_install.sh
   ```
   *(Скрипт скопирует скомпилированный `paste_helper_type` в `/tmp/` для быстрого доступа).*

3. Запустите приложение:
   ```bash
   python3 DictationApp.py
   ```

## Важно: Права Accessibility (Универсальный доступ)
Для того чтобы приложение могло перехватывать глобальное нажатие `Cmd+D` и само печатать текст, ему нужны права Универсального доступа macOS.

1. Зайдите в **Системные настройки (System Settings)** -> **Конфиденциальность и безопасность (Privacy & Security)** -> **Универсальный доступ (Accessibility)**.
2. Дайте разрешение терминалу, из которого вы запускаете скрипт (например, iTerm, Terminal или Cursor).
3. **ВНИМАНИЕ Если ввод сломался:** Права жестко привязаны к подписи бинарного файла (например, системного `python3` или вашего терминала). Если вы обновили Python, перешли на `uv run`, или перекомпилировали Swift-файл, macOS **молча отзовет права на имитацию клавиш**. Текст будет копироваться в буфер, но не печататься. 
   - **Решение:** В настройках Универсального доступа удалите приложение (кнопкой `-`) и добавьте его заново (кнопкой `+`).

## Как обернуть в `.app` (Опционально)
Чтобы не держать терминал открытым, можно запаковать скрипт в macOS `.app`:
1. Создайте структуру папок: `Dictation.app/Contents/MacOS/`.
2. Внутрь положите скрипт-лаунчер `DictationLauncher`:
   ```bash
   #!/bin/bash
   export PATH="$HOME/Library/Python/3.9/bin:$PATH"
   exec /usr/bin/python3 "/путь/к/DictationApp.py"
   ```
3. Сделайте его исполняемым (`chmod +x`). 
4. Теперь вы можете запускать диктовку как обычное приложение (не забудьте выдать самому `.app` права Accessibility).
